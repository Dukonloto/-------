Заключение

В данной проектной работе изучен механизм Клиентсерверного взаимодействия на примере протокола HTTP.

Протокол HTTP является протоколом верхнего уровня и описывает порядок получения клиентом документов от сервера.

Обычно клиентом является web браузер, но клиентом может выступать любая программа как например telnet или netcat необходимо просто знать правила взаимодействия клиента и сервера.

Кроме того, изучены протоколы TCP/IP с помощью инструмента wireshark так как протокол HTTP является протоколом уровня приложений то ниже уровнем находится протокол TCP что наглядно видно в инструменте wireshark.

Еще одним объектом изучения была серверная часть на примере программного обеспечения Nginx.

Web сервер умеет предоставлять клиенту как статичные файлы или коды ответов неудачного поиска так и динамические документы, выступая Proxy сервером для самописаных web приложений.

Выводы

• Протокол HTTP является открытым протоколом.

• Имеется обширный набор инструментов для диагностики работы HTTP протокола.

• HTTP протокол позволяет передавать не только статичные файлы но и динамические.

• Динамические HTTP документы генерируются “на лету” web приложениями.

Технология формирования динамических документов в настоящее время имеет широкое применение в мобильных приложениях, ботах социальных сетей, интернет магазинах и крипто-торговли.

Считается что объединение динамических HTTP страниц, крипто технологий и искусственного интеллекта создаст новое поколение web которое получит название WEB3.
